<objective>
Make `bun test tests/` order-independent and 0-fail by removing the single proven poisoner (TEST-01), and add a durable guard that stops the same class of leak from returning (TEST-02).

Purpose: bun's `mock.module(...)` is process-global and is NOT undone by `mock.restore()` (oven-sh/bun#7823). `tests/unit/ttl-calculator-actions.test.ts::setupCommonMocks()` partial-mocks pure shared modules (`@/lib/utils` dropping `cn`, `@/lib/constants/business` dropping `links.facebook`, plus a `@/lib/schemas/ttl` stub) and redundantly re-mocks `@/env`/`@/lib/logger` that `tests/setup.ts` already mocks completely. Those leaked partial exports freeze the shared modules for every later test file in the run, producing ~21 homepage/navigation/Footer failures (dominant symptom: `SyntaxError: Export named 'cn' not found`). This blocks Phase 18 because the Dependabot PRs' CI Test job hits the same pollution.

Output: a reduced mock helper (only the real boundaries `@/lib/db` + `@/lib/resend-client` stay mocked), a `scripts/check-test-mock-leaks.sh` denylist guard wired into both the local `test:unit` script and the CI Code Quality job, and a short TESTING convention note in `tests/setup.ts`.

This is TEST-INFRA ONLY. No `src/**` runtime file is modified. The real `@/lib/utils`, `@/lib/constants/business`, and `@/lib/schemas/ttl` modules already work as-is for this action; no `src` change is required.
</objective>

|----------|-------------|
| test file -> process-global module registry | A per-file `mock.module(...)` mutates a registry shared by every later test file (bun#7823); a partial mock is an untrusted write that freezes a module's exports suite-wide. |
| CI signal -> merge decision | The Test job's pass/fail count gates merges; a polluted suite makes the signal untrustworthy (the v7 milestone reason this phase exists). |
