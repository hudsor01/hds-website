## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public client -> /api/calculators/submit | Untrusted JSON body crosses here (passes CSRF/rate-limit); arbitrary inputs/results persisted to JSON columns |
| cron/n8n -> /api/process-emails (GET/POST) | Bearer-authenticated; overlapping invocations race on the shared pending-email rows |
| admin client -> /api/testimonials/[id] | Bearer-authenticated mutation; id is a path param (untrusted shape) reaching a uuid column |
| app instance -> Upstash Redis | Network boundary; runtime failure forces in-memory fallback (unbounded growth risk) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-01 | Repudiation/Tampering | processPendingEmails double-send race | mitigate | Atomic claim (UPDATE ... status='processing' WHERE status='pending' RETURNING) before send; send only claimed rows (Task 1) |
| T-20-02 | Denial of Service | rate-limiter in-memory fallback unbounded under Redis outage | mitigate | Lazy prune of expired entries in _checkLimitInMemory on every call, independent of useRedis (Task 2) |
| T-20-03 | Tampering | rate-limiter Redis non-atomic incr/expire -> TTL-less zombie key | mitigate | Atomic count+TTL via pipeline/multi (or SET NX EX + INCR) (Task 2) |
| T-20-04 | Information Disclosure | testimonials/[id] 500 leaking Postgres 22P02 on malformed id | mitigate | Route validates id with z.string().uuid() -> 400 before DB; not-found -> 404 (Task 3) |
| T-20-05 | Denial of Service | calculators/submit unbounded JSON -> storage bloat / payload DoS | mitigate | Serialized-size + key-count cap before insert -> 400 (Task 4) |
| T-20-SC | Tampering | npm/pip/cargo installs | mitigate | Default no-new-dep; @upstash/ratelimit only on explicit operator approval via checkpoint:decision (Task 5). No [ASSUMED]/[SUS] installs planned. |
</threat_model>
