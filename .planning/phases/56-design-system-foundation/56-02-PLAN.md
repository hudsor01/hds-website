**Fix 1: Broken font variable in globals.css**

The `@theme {}` block has:
```css
--font-sans: var(--font-spline), system-ui, sans-serif;
```
But `layout.tsx` loads `Geist` with `variable: '--font-geist-sans'` — there is no `--font-spline` variable injected anywhere. The body font silently falls back to `system-ui`.

In `globals.css` `@theme {}`, change:
```css
--font-sans: var(--font-spline), system-ui, sans-serif;
```
to:
```css
--font-sans: var(--font-geist-sans), system-ui, sans-serif;
--font-mono: var(--font-geist-mono), ui-monospace, monospace;
```

This wires the Next.js `next/font/google` CSS variables (injected by layout.tsx onto `<html>`) to the Tailwind font-sans token.

**Fix 2: Add fluid type scale tokens to @theme {}**

After the `/* Radius */` group in `@theme {}`, add a `/* Type scale */` group:

```css
/* Type scale — fluid via clamp(), mobile-first */
--font-size-xs:    clamp(0.75rem, 1.5vw, 0.875rem);
--font-size-sm:    clamp(0.875rem, 1.8vw, 1rem);
--font-size-base:  clamp(1rem, 2vw, 1.125rem);
--font-size-lg:    clamp(1.125rem, 2.5vw, 1.25rem);
--font-size-xl:    clamp(1.25rem, 3vw, 1.5rem);
--font-size-2xl:   clamp(1.5rem, 3.5vw, 1.875rem);
--font-size-3xl:   clamp(1.875rem, 4vw, 2.25rem);
--font-size-4xl:   clamp(2.25rem, 5vw, 3rem);
--font-size-hero:  clamp(2.5rem, 6vw, 4rem);

/* Font weights */
--font-weight-normal:   400;
--font-weight-medium:   500;
--font-weight-semibold: 600;
--font-weight-bold:     700;
--font-weight-heading:  700;

/* Line heights */
--line-height-tight:    1.1;
--line-height-snug:     1.25;
--line-height-normal:   1.5;
--line-height-relaxed:  1.625;
--line-height-hero:     1.1;

/* Letter spacing */

--tracking-normal:   0em;
--tracking-wide:     0.025em;
--tracking-widest:   0.1em;
--tracking-heading: -0.02em;
```

These become Tailwind utilities: `text-size-hero`, `font-weight-heading`, `leading-hero`, `tracking-heading` etc. Tailwind v4 auto-generates utility classes from `@theme {}` tokens.

**Add shadow scale tokens to @theme {}:**

After the type scale group, add a `/* Shadow scale */` group:

```css
/* Shadow scale */
--shadow-xs:   0 1px 2px oklch(0 0 0 / 0.04);
--shadow-sm:   0 1px 3px oklch(0 0 0 / 0.06), 0 1px 2px oklch(0 0 0 / 0.04);
--shadow-md:   0 4px 12px oklch(0 0 0 / 0.08), 0 2px 4px oklch(0 0 0 / 0.04);
--shadow-lg:   0 8px 24px oklch(0 0 0 / 0.10), 0 4px 8px oklch(0 0 0 / 0.06);
--shadow-xl:   0 20px 40px oklch(0 0 0 / 0.12), 0 8px 16px oklch(0 0 0 / 0.08);
--shadow-glow: 0 0 30px oklch(from var(--color-primary) l c h / 0.20);
--shadow-glow-accent: 0 0 20px oklch(from var(--color-accent) l c h / 0.25);
```

Note: Use plain OKLCH black `oklch(0 0 0 / alpha)` for shadow base color — this works correctly in both light and dark mode without needing separate dark shadow tokens. The opacity provides the depth effect.
