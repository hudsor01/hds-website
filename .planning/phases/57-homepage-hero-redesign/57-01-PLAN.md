Create `tests/unit/homepage.test.tsx` using bun:test + @testing-library/react. The test file renders the `HomePage` component from `@/app/page.tsx` and asserts structural properties required by HERO-01 through HERO-04.

Test structure:
- Import: `import { describe, expect, it } from 'bun:test'`
- Import: `import { render } from '@testing-library/react'`
- Import: `import HomePage from '@/app/page.tsx'` (default export)

Note: `src/app/page.tsx` is a Server Component that uses metadata export — @testing-library/react renders it as a synchronous function in happy-dom. No 'use client' needed.

Tests to implement (group under `describe('HomePage structural assertions')`):

**HERO-02 — Headline hierarchy:**
- `it('h1 has text-page-title class', ...)` — render, find h1, assert `className` contains `text-page-title`
- `it('supporting paragraph has text-muted-foreground class', ...)` — find the `<p>` element that is a direct sibling after h1 in the hero, assert `className` contains `text-muted-foreground`
- `it('no span inside h1 has text-accent class', ...)` — find h1, get all descendant spans via `querySelectorAll('span')`, assert none have className containing `text-accent`

**HERO-03 — CTA differentiation:**
- `it('primary CTA button has bg-accent class', ...)` — find the first link button in the hero CTA group, assert the rendered anchor or button element has a class containing `bg-accent`
- `it('secondary CTA button does not have bg-accent class', ...)` — find the second link button in the hero CTA group, assert className does NOT contain `bg-accent`

**HERO-01 — Background treatment:**
- `it('hero section does not use a CSS gradient background', ...)` — query all `<section>` elements, assert none have className containing the string `gradient`
- `it('hero section has dark background class', ...)` — find the first `<section>` (the hero), assert its className contains `bg-background-dark` OR `bg-[var(--color-background-dark)]` OR `bg-[oklch`

**HERO-04 — Section rhythm:**
- `it('no section element has bg-muted class', ...)` — query all `<section>` elements, assert none have className containing `bg-muted`
- `it('no element has scale-105 class', ...)` — use `container.querySelectorAll('[class*="scale-105"]')` or iterate all elements; assert none have class containing `scale-105`
- `it('no element has animationDelay inline style', ...)` — use `container.querySelectorAll('[style]')` and assert none contain `animationDelay`

Use `render(<HomePage />)` to get `container`. Query using `container.querySelector()` / `container.querySelectorAll()`. Use `Array.from()` to iterate NodeLists.

Note: These tests will FAIL against the current page.tsx — that is expected. They turn green after Wave 2-3 implementation. Do not skip or mark as expected-to-fail.

Follow patterns from `tests/setup.ts` (mocks `@/env` and `@/lib/logger` globally). No additional mocks needed for this file.
