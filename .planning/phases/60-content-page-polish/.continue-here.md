### 2. Showcase card redesign
Current problems (confirmed via screenshots):
- Featured cards (md:col-span-2) have a huge dark color header panel (h-80) with text clustered in the top-left corner — "LOCAL BUSINESS" eyebrow top-left, title middle-left, "Case Study" badge bottom-left. Massive wasted space.
- Non-featured cards (half-width) have the same layout at h-64 — same clutter problem at smaller scale.
- The header panel uses `justify-between` on a flex-col, spreading 3 small text elements across a tall panel — this creates the "cluttered top-left, empty right side" look.
- Below the header: description (2-line clamp), metrics row, tech stack chips — all fine but the header dominates.

Card redesign approach options:
A) **Reduce header height dramatically** (h-48/h-36), center the title, move category to a small badge — less wasted dark space
B) **Horizontal layout for featured cards** — image/color panel on left (40%), content on right (60%) — eliminates the tall stacked layout
C) **Remove the large color panel entirely** — use a subtle accent border-left or top accent bar (like the stats grid), put all content in a clean card layout with category badge, title, description, metrics, tech stack
D) **Keep panel but make it useful** — add a subtle pattern/illustration, or make the panel shorter with centered text

The user's complaint about "text all jumbled up and cluttered in the top left corner" points strongly toward option C — eliminating the large color panel in favor of a cleaner, content-first card design.
