    **lefthook.yml (WKFL-02):**

    Read the existing lefthook.yml (currently has `lint`, `typecheck`, and `test` in pre-commit, and `lint`, `typecheck`, `test` in pre-push).

    Per CONTEXT.md decisions:
    - Replace the `lint` command in pre-commit with Biome's staged-files check (do NOT auto-fix, block only)
    - Keep `typecheck` and `test` commands in pre-commit as-is (they are not ESLint-related)
    - Remove the entire `pre-push` section (CONTEXT.md: "No pre-push hook — pre-commit is sufficient")

    Write the new lefthook.yml:
    ```yaml
    # Lefthook configuration
    # Documentation: https://github.com/evilmartians/lefthook

    pre-commit:
      parallel: true
      commands:
        check:
          glob: "*.{js,ts,cjs,mjs,d.cts,d.mts,jsx,tsx,json,jsonc}"
          run: bunx biome check --no-errors-on-unmatched --files-ignore-unknown=true --colors=off {staged_files}
        typecheck:
          glob: "*.{ts,tsx}"
          run: bun run typecheck
        test:
          run: bun run test:unit
    ```

    Key points:
    - Use `{staged_files}` (lefthook variable), NOT `--staged` (Biome's own flag) — per official Biome docs
    - Include `--no-errors-on-unmatched` to prevent false failures on .md-only commits
    - Include `--files-ignore-unknown=true` for non-JS/TS file safety
    - Use `bunx biome` (project uses Bun, not npx)
    - `parallel: true` preserved from existing config

    After writing lefthook.yml, reinstall hooks:
    ```bash
    bun run prepare
    ```

    Stage and commit:
    ```bash
    git add package.json lefthook.yml
    git commit -m "chore(54): migrate lint/format scripts and hooks to biome"
    ```
