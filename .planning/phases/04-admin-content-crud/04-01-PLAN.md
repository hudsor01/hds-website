  <behavior>
    `requireAdminSession()`:
    - Calls `getSession()`; if `null`, calls `redirect('/auth/sign-in')`.
    - If `session.user.role !== 'admin'`, calls `redirect('/admin')` (the layout will then render `<Forbidden />`).
    - Otherwise returns the non-null `SessionData` (narrowed so callers can use `.user.id` without ?.).

    `slugify(input: string): string`:
    - Lowercases, NFKD-normalizes and strips combining diacritics (`input.normalize('NFKD').replace(/\p{M}/gu, '')`).
    - Replaces every run of non-[a-z0-9] chars with a single `-`.
    - Trims leading and trailing `-`.
    - Returns `''` for input that has no alphanumeric chars.
    - Tests: `slugify('Hello, World!')` → `'hello-world'`; `slugify('Café & Té')` → `'cafe-te'`; `slugify('  --A  B  --')` → `'a-b'`; `slugify('!!!')` → `''`; `slugify('JirahShop e-commerce')` → `'jirahshop-e-commerce'`.

    `formDataToObject(formData: FormData): Record<string, string | string[]>`:
    - Iterates `formData.entries()`. If a key appears more than once, the value becomes a `string[]`; otherwise a single `string`. File values are stringified via `String(value)` (admin forms never upload files — D-04 — but the helper must not crash if they do).
    - Tests: single key → string value; repeated key (tag multi-select) → string array; empty FormData → `{}`.

    `isUniqueViolation(error: unknown, column: string): boolean`:
    - True iff `error` is an object with `error.code === '23505'` AND its `message` (or `detail`) contains the literal `column` substring.
    - Returns false for any non-Postgres error or any constraint not matching `column`.
    - Tests: `{ code: '23505', message: 'duplicate key value violates unique constraint "showcase_slug_unique" ... Key (slug)=...' }` with column `'slug'` → true; same error with column `'title'` → false; plain `Error('boom')` → false; `null` → false.
  </behavior>

    export type AdminSession = NonNullable<SessionData> & {
      user: NonNullable<SessionData>['user'] & { role: 'admin' }
    }

    export async function requireAdminSession(): Promise<AdminSession> {
      const session = await getSession()
      if (!session?.user) {
        redirect('/auth/sign-in')
      }
      if (session.user.role !== 'admin') {
        redirect('/admin')
      }
      return session as AdminSession
    }

    ## 3. src/lib/admin/form-data.ts

    Pure. Export:

    ```ts
    export function formDataToObject(
      formData: FormData
    ): Record<string, string | string[]> {
      const out: Record<string, string | string[]> = {}
      for (const [key, value] of formData.entries()) {
        const v = typeof value === 'string' ? value : String(value)
        if (key in out) {
          const existing = out[key]
          out[key] = Array.isArray(existing) ? [...existing, v] : [existing as string, v]
        } else {
          out[key] = v
        }
      }
      return out
    }
    ```

    ## 4. src/lib/admin/db-errors.ts

    Pure. Export:

    ```ts
    export function isUniqueViolation(error: unknown, column: string): boolean {
      if (!error || typeof error !== 'object') return false
      const e = error as { code?: unknown; message?: unknown; detail?: unknown }
      if (e.code !== '23505') return false
      const text = `${typeof e.message === 'string' ? e.message : ''} ${typeof e.detail === 'string' ? e.detail : ''}`
      return text.includes(column)
    }
    ```

    Matches Postgres `unique_violation` SQLSTATE 23505 per `node-postgres` error shape; the slug constraint message includes the column name (e.g. `Key (slug)=(...)` in `detail`).

    ## 1. src/components/admin/FormFieldSet.tsx (server component)

    Labeled wrapper for form fields used INSIDE useAppForm field renderers (so the field hook context is already established by the caller). Mirrors the aria pattern from `GenericField` in `src/hooks/form-hook.tsx` but without binding to TanStack so a per-resource field can decide its own input element.

    ```tsx
    import type { ReactNode } from 'react'

    interface FormFieldSetProps {
      label: string
      htmlFor: string
      required?: boolean
      error?: string
      hint?: string
      children: ReactNode
    }

    export function FormFieldSet({
      label,
      htmlFor,
      required,
      error,
      hint,
      children
    }: FormFieldSetProps) {
      const errorId = error ? `${htmlFor}-error` : undefined
      const hintId = hint ? `${htmlFor}-hint` : undefined
      return (
        <div className="space-y-1.5">
          <label htmlFor={htmlFor} className="block text-sm font-medium text-foreground">
            {label}
            {required && (
              <>
                <span className="text-destructive" aria-hidden="true">{' *'}</span>
                <span className="sr-only"> required</span>
              </>
            )}
          </label>
          {children}
          {hint && (
            <p id={hintId} className="text-xs text-muted-foreground">{hint}</p>
          )}
          {error && (
            <p id={errorId} role="alert" className="text-xs text-destructive">{error}</p>
          )}
        </div>
      )
    }
    ```

    The child input is responsible for setting `id={htmlFor}`, `aria-describedby={errorId ?? hintId}`, `aria-invalid={error ? 'true' : undefined}`.

    ## 2. src/components/admin/DeleteButton.tsx (`'use client'`)

    Renders a `<form>` whose `action` is a Server Action (passed via prop) and includes a hidden `id` field. Click intercept calls `window.confirm()` with the resource label; cancel aborts the submit.

    ```tsx
    'use client'

    import { Trash2 } from 'lucide-react'

    interface DeleteButtonProps {
      // The deleteAction is a Server Action; React's typing for this is currently
      // a regular function reference passed to <form action={...}>.
      action: (formData: FormData) => void | Promise<void>
      id: string
      label: string  // e.g. "Delete showcase"
      confirmMessage: string  // e.g. 'Delete "JirahShop e-commerce"? This cannot be undone.'
    }

    export function DeleteButton({ action, id, label, confirmMessage }: DeleteButtonProps) {
      return (
        <form action={action}>
          <input type="hidden" name="id" value={id} />
          <button
            type="submit"
            onClick={(e) => {
              if (!window.confirm(confirmMessage)) {
                e.preventDefault()
              }
            }}
            className="inline-flex items-center gap-2 px-3 py-2 rounded-md text-sm font-medium text-destructive border border-destructive/40 hover:bg-destructive/10 transition-smooth"
          >
            <Trash2 size={16} aria-hidden="true" />
            {label}
          </button>
        </form>
      )
    }
    ```

    Per CONTEXT.md D-08: native `window.confirm` is sufficient for v1 (no custom modal).
