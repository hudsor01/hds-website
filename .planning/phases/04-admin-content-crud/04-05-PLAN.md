The Next.js App Router `(coming-soon)` is a route GROUP (parentheses), so the URL paths for the stubs are `/admin/showcase`, `/admin/blog`, etc. The SAME paths as the new routes added in Waves 1-2.

When two segments resolve to the same URL, Next.js throws a build error ("You cannot have two parallel pages that resolve to the same path"). This is precisely WHY the stub deletion is its own plan AFTER Wave 2: each Wave 2 plan was instructed NOT to delete the stub, and each Wave 2 verification step that ran `bun run build` is therefore expected to PRODUCE this parallel-pages error. Plans 04-02, 04-03, 04-04 verification gates intentionally use `bun run typecheck` + `bun run lint` (NOT `bun run build`) for that reason. Plan 04-06 (final verification) is the first place a clean `bun run build` is required.

CORRECTION: re-reading the Next.js route-group semantics, two files under different route groups resolving to the same URL DO cause a build error. To keep `bun run build` available as a verification gate during Waves 1-2, plans 04-02 through 04-04 deliberately scope their build verification to typecheck + lint only. Plan 04-05 (this plan) is the first wave where `bun run build` MUST succeed.
