# Requirements: v3.1 Biome Migration

**Defined:** 2026-02-24
**Core Value:** Working tools and contact form stay functional while the codebase achieves production-grade quality: strict types, comprehensive test coverage, proper error handling, visual correctness, and accessible to all users.

## v3.1 Requirements

### Biome Setup

- [ ] **BIOM-01**: Developer can run `biome check .` to lint and format-check the entire codebase (replacing `eslint .`)
- [ ] **BIOM-02**: Biome formatter replaces Prettier — all JS/TS/JSON files formatted consistently via `biome format`
- [ ] **BIOM-03**: React and Next.js lint rules enforced via Biome's built-in domains (no ESLint plugins needed)
- [ ] **BIOM-04**: TypeScript unused-var rules fully covered: `noUnusedVariables`, `noUnusedFunctionParameters`, `noUnusedImports`
- [ ] **BIOM-05**: Curly-braces rule enforced via `useBlockStatements` (matching current ESLint `curly: all`)
- [ ] **BIOM-06**: Tailwind CSS directives (`@layer`, `@apply`, `@tailwind`) do not trigger false Biome lint errors

### Developer Workflow

- [x] **WKFL-01**: `package.json` lint and format scripts invoke Biome (not ESLint/Prettier)
- [x] **WKFL-02**: Lefthook pre-commit hook lints staged files only with Biome (replaces slow full-repo ESLint scan)
- [x] **WKFL-03**: CI workflow runs Biome check (ESLint step renamed/replaced)
- [x] **WKFL-04**: VSCode settings use Biome extension for format-on-save

### Cleanup

- [x] **CLEN-01**: `eslint`, `eslint-config-next`, and `prettier` removed from devDependencies
- [x] **CLEN-02**: `eslint.config.mjs`, `.prettierrc.json`, `.prettierignore` deleted from repository
- [x] **CLEN-03**: Codebase reformatted by Biome in an isolated commit (preserves `git blame` usefulness)

### Zero Regression

- [x] **ZERO-01**: All 329+ unit tests continue to pass after migration
- [x] **ZERO-02**: Production build generates all 129 static pages without errors
- [x] **ZERO-03**: `tsc --noEmit` reports 0 TypeScript errors after migration

## Out of Scope

| Feature | Reason |
|---------|--------|
| Hybrid ESLint + Biome approach | Research confirms full coverage; no hybrid needed |
| Biome CSS formatting | CSS formatter is experimental in Biome 2.x; CSS files excluded |
| Replacing TypeScript type checking | Biome does not run type-aware rules; `tsc --noEmit` is retained |
| Adding new lint rules beyond current coverage | Scope is parity migration, not rule expansion |
