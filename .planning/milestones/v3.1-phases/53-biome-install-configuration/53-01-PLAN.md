```bash

```

Then replace the content of biome.json with the complete hand-crafted configuration below. Do NOT run `biome migrate eslint` — it will fail with the eslint-config-next/@rushstack/eslint-patch error (confirmed in RESEARCH.md as known issue #2935, closed-not-planned).

Key decisions baked in (from RESEARCH.md and CONTEXT.md locked decisions):
- `arrowParentheses: "asNeeded"` — matches .prettierrc.json `"arrowParens": "avoid"` (user deferred to Claude; "asNeeded" minimizes Phase 54 diff)
- `noNonNullAssertion: "error"` — escalated from ESLint "warn" per project fail-fast principle
- `noConsole` with `allow: ["warn", "error"]` — matches project's `noConsoleLog` intent (not noConsole globally)
- `files.includes` with `!!` prefix — Biome 2.x syntax for hard directory exclusions (not old `files.ignore`)
- `css.parser.tailwindDirectives: true` — required for Tailwind v4 at-rules (@theme, @source, @utility)
- All rules set to "error" severity — no "warn" allowed (fail-fast principle)
