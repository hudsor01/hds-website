## v3.1 Biome Migration (Phases 53-55)

**Milestone Goal:** Replace ESLint + Prettier with Biome — single Rust binary, zero npm vulnerability surface, full TypeScript/React rule parity. Zero regression on all tests and build.

### Phases

- [x] **Phase 53: Biome Install & Configuration** - Install @biomejs/biome and produce a fully tuned biome.json covering all lint and format rules (completed 2026-02-24)
- [x] **Phase 54: Format Sweep & Workflow Integration** - Reformat the codebase in an isolated commit, then wire Biome into package.json scripts, lefthook hooks, CI, and VSCode (completed 2026-02-25)
- [x] **Phase 55: Old Tooling Removal & Verification** - Remove ESLint and Prettier packages and config files, confirm zero regression across tests, build, and type checking (completed 2026-02-25)

## Phase Details

### Phase 53: Biome Install & Configuration
**Goal**: Developer can run `biome check .` and get full lint + format coverage equivalent to the current ESLint + Prettier setup
**Depends on**: Phase 52 (clean baseline — all 329+ tests passing, 0 TypeScript errors, 0 lint errors)
**Requirements**: BIOM-01, BIOM-02, BIOM-03, BIOM-04, BIOM-05, BIOM-06
**Success Criteria** (what must be TRUE):
  1. `bunx biome check src/` completes with 0 errors and 0 warnings on the unformatted codebase
  2. `biome.json` explicitly configures `noUnusedVariables`, `noUnusedFunctionParameters`, and `noUnusedImports` in the `correctness` group
  3. `biome.json` explicitly configures `useBlockStatements: "error"` in the `style` group (curly braces on all if/else)
  4. React and Next.js lint domains are enabled in `biome.json` and fire on a test violation
  5. Tailwind CSS directives (`@tailwind`, `@layer`, `@apply`) in `globals.css` produce zero Biome lint errors
**Plans**: 1 plan

Plans:
- [ ] 53-01-PLAN.md — Install Biome 2.4.4, hand-craft biome.json, verify 0 errors/warnings with rule gap checkpoint

### Phase 54: Format Sweep & Workflow Integration
**Goal**: The entire codebase is formatted by Biome in a single isolated commit, and all developer tooling (scripts, hooks, CI, editor) invokes Biome instead of ESLint/Prettier
**Depends on**: Phase 53 (biome.json finalized before reformatting)
**Requirements**: CLEN-03, WKFL-01, WKFL-02, WKFL-03, WKFL-04
**Success Criteria** (what must be TRUE):
  1. A single standalone commit exists with message `chore: reformat codebase with biome` containing only formatting changes (no config, no logic)
  2. `bun run lint` invokes `biome check` (not `eslint`) and exits 0 on the formatted codebase
  3. `bun run format` invokes `biome format --write` (not Prettier)
  4. `git commit` on a staged file with a lint error is blocked by the lefthook pre-commit hook running Biome
  5. `.vscode/settings.json` configures the Biome extension as the default formatter for JS/TS/JSON files
**Plans**: 1 plan
