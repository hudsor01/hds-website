## E2E Flows

### Flow 1: Token Cascade → Page Render

1. `globals.css @theme {}` defines OKLCH tokens → Tailwind v4 generates utilities
2. `layout.tsx` injects Geist Sans CSS variable via `<html>` className
3. `page.tsx` applies `text-page-title`, `py-section`, `bg-surface-raised`, `hero-spotlight` etc.
4. Browser resolves CSS custom properties → visually distinct design system

**Status: COMPLETE**

### Flow 2: Tool Page Render (12 of 13 tools)

1. User visits a tool page (e.g. `/tools/json-formatter`)
2. Server page.tsx renders `<JsonFormatterClient />`
3. `ToolPageLayout` renders header (`text-h1` + `text-lead` + Phase 56 tokens)
4. `Card variant="glassLight"` wraps result pane (Phase 58 → Phase 56 tokens)
5. `Button variant="ghost"` renders in action bar (Phase 58)
6. 408 tests pass; TypeScript + Biome clean

**Status: COMPLETE**

### Flow 3: PDF Tool Download

1. User visits Contract/Invoice/Proposal Generator
2. Fills required fields → `hasResult` gate opens → `ResultCard` renders
3. Action bar shows "Download PDF" button (`ToolPageLayout actions` prop)
4. Click triggers `handleDownload` → `pdf(<Document />).toBlob()` → file download

**Status: COMPLETE (programmatic blob pattern)**

### Flow 4: TTL Calculator Render

1. User visits `/tools/ttl-calculator`
2. Route renders legacy `<Calculator />` — no `ToolPageLayout` wrapper
3. Calculator component renders own `InputPanel` / `ResultsPanel` layout
4. Header manually uses `text-h1` (left-aligned, no icon) — matches Phase 59 style spec

**Status: PARTIALLY COMPLETE** — Calculator functions correctly; layout/card architecture differs from other 12 tools. Header manually aligned per VERIFICATION.md Truth 14.
