## n8n Workflow Import

### 1. Access n8n Dashboard

```bash
# Get n8n URL from Traefik ingress
kubectl get ingress -n automation

# Or port-forward n8n
kubectl port-forward -n automation svc/n8n 5678:5678

# Open browser: http://localhost:5678 or https://n8n.yourdomain.com
```

### 2. Create Credentials

#### PostgreSQL (Supabase) - Required for All Workflows

1. Go to **Settings** > **Credentials** > **New**
2. Select **Postgres**
3. Enter connection details:
   - Name: `Supabase PostgreSQL`
   - Host: `your-supabase-host.supabase.co`
   - Database: `postgres`
   - User: `postgres` (or `n8n_user`)
   - Password: `your_password`
   - Port: `5432`
   - SSL: `require`
4. Test connection and save

#### Resend API - Required for Workflows 01, 02, 06, 07, 10, 12

1. Go to **Settings** > **Credentials** > **New**
2. Select **HTTP Header Auth**
3. Enter:
   - Name: `Resend API Key`
   - Header Name: `Authorization`
   - Value: `Bearer re_your_resend_api_key_here`
4. Save

#### Redis - Required for Workflows 09, 11

1. Go to **Settings** > **Credentials** > **New**
2. Select **Redis**
3. Enter:
   - Name: `Redis`
   - Host: Your Redis host
   - Port: `6379`
   - Database: `0`
   - Password: Your Redis password (if applicable)
4. Test connection and save

#### Ghost Admin API - Required for Workflow 05

1. Go to **Settings** > **Credentials** > **New**
2. Select **HTTP Header Auth**
3. Enter:
   - Name: `Ghost Admin API Key`
   - Header Name: `Authorization`
   - Value: `Ghost your_ghost_admin_api_key_here`
4. Save

#### Twitter OAuth2 - Required for Workflow 08

1. Go to **Settings** > **Credentials** > **New**
2. Select **OAuth2 API**
3. Name: `Twitter OAuth2`
4. Configure with Twitter Developer Portal credentials:
   - Client ID: Your Twitter app client ID
   - Client Secret: Your Twitter app client secret
   - Auth URL: `https://twitter.com/i/oauth2/authorize`
   - Access Token URL: `https://api.twitter.com/2/oauth2/token`
   - Scope: `tweet.read tweet.write users.read`
5. Authenticate and save

#### LinkedIn OAuth2 - Required for Workflow 08

1. Go to **Settings** > **Credentials** > **New**
2. Select **OAuth2 API**
3. Name: `LinkedIn OAuth2`
4. Configure with LinkedIn Developer credentials:
   - Client ID: Your LinkedIn app client ID
   - Client Secret: Your LinkedIn app client secret
   - Auth URL: `https://www.linkedin.com/oauth/v2/authorization`
   - Access Token URL: `https://www.linkedin.com/oauth/v2/accessToken`
   - Scope: `w_member_social`
5. Authenticate and save

#### Facebook OAuth2 - Required for Workflow 08

1. Go to **Settings** > **Credentials** > **New**
2. Select **OAuth2 API**
3. Name: `Facebook OAuth2`
4. Configure with Facebook Developer credentials:
   - Client ID: Your Facebook app ID
   - Client Secret: Your Facebook app secret
   - Auth URL: `https://www.facebook.com/v18.0/dialog/oauth`
   - Access Token URL: `https://graph.facebook.com/v18.0/oauth/access_token`
   - Scope: `pages_manage_posts pages_read_engagement`
5. Authenticate and save

#### GitHub Token - Required for Workflow 12

1. Go to **Settings** > **Credentials** > **New**
2. Select **HTTP Header Auth**
3. Enter:
   - Name: `GitHub Token`
   - Header Name: `Authorization`
   - Value: `token ghp_your_github_personal_access_token`
4. Save

**Generate GitHub Personal Access Token:**
1. Go to GitHub Settings > Developer settings > Personal access tokens
2. Generate new token (classic)
3. Select scopes: `repo` (full control of private repositories)
4. Copy token and use in n8n credential

### 2. Contact Form Integration (Workflows 01, 11)

Edit `src/app/api/contact/route.ts`:

```typescript
import { env } from '@/env';
import { logger } from '@/lib/logger';

export async function POST(request: Request) {
  const formData = await request.json();

  // Step 1: Spam validation (workflow 11)
  const spamCheck = await fetch(env.N8N_SPAM_VALIDATOR_WEBHOOK, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      email: formData.email,
      firstName: formData.firstName,
      lastName: formData.lastName,
      phone: formData.phone,
      company: formData.company,
      message: formData.message,
      honeypot_field: formData.honeypot || '',
      form_interaction_time: formData.interactionTime || 0,
      user_agent: request.headers.get('user-agent') || '',
    }),
  });

  const spamResult = await spamCheck.json();

  // Reject if classified as spam
  if (spamResult.action === 'reject') {
    logger.warn('Form submission blocked as spam', { email: formData.email });
    return new Response(
      JSON.stringify({
        error: 'Submission blocked. Please contact us directly.'
      }),
      { status: 400 }
    );
  }

  // Step 2: Lead capture with AI qualification (workflow 01)
  try {
    const leadCapture = await fetch(env.N8N_LEAD_CAPTURE_WEBHOOK, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        firstName: formData.firstName,
        lastName: formData.lastName,
        email: formData.email,
        company: formData.company,
        phone: formData.phone,
        message: formData.message,
        budget: formData.budget,
        timeline: formData.timeline,
        source: 'website-contact-form',
      }),
    });

    const result = await leadCapture.json();
    logger.info('Lead captured successfully', {
      leadId: result.leadId,
      spamScore: spamResult.spam_score
    });

    return new Response(
      JSON.stringify({ success: true, leadId: result.leadId }),
      { status: 200 }
    );
  } catch (error) {
    logger.error('Failed to capture lead', error as Error);
    return new Response(
      JSON.stringify({ error: 'Failed to process submission' }),
      { status: 500 }
    );
  }
}
```

### 3. Attribution Tracking (Workflow 09)

Create middleware for attribution tracking `src/middleware.ts`:

```typescript
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';
import { env } from '@/env';

export async function middleware(request: NextRequest) {
  const response = NextResponse.next();

  // Track attribution data on every page load
  const url = new URL(request.url);

  // Only track on main site pages (not API routes or static assets)
  if (!url.pathname.startsWith('/api') && !url.pathname.startsWith('/_next')) {
    // Fire and forget attribution tracking
    fetch(env.N8N_ATTRIBUTION_TRACKER_WEBHOOK, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        page_url: request.url,
        page_title: url.pathname,
        referrer: request.headers.get('referer') || '',
        user_agent: request.headers.get('user-agent') || '',
        // UTM parameters will be parsed from URL by n8n workflow
        utm_source: url.searchParams.get('utm_source'),
        utm_medium: url.searchParams.get('utm_medium'),
        utm_campaign: url.searchParams.get('utm_campaign'),
        utm_term: url.searchParams.get('utm_term'),
        utm_content: url.searchParams.get('utm_content'),
      }),
    }).catch(() => {
      // Silently fail - don't block page load
    });
  }

  return response;
}

export const config = {
  matcher: [
    '/((?!api|_next/static|_next/image|favicon.ico).*)',
  ],
};
```

### Monitor Workflow Performance

```sql
-- Lead qualification success rate
SELECT
  COUNT(*) as total_leads,
  AVG(qualification_score) as avg_score,
  COUNT(CASE WHEN qualification_score >= 75 THEN 1 END) as hot_leads
FROM leads;

-- Invoice generation metrics
SELECT
  COUNT(*) as total_invoices,
  SUM(total_amount) as total_invoiced,
  AVG(total_amount) as avg_invoice_amount
FROM invoices
WHERE created_at >= NOW() - INTERVAL '30 days';

-- Attribution channel breakdown
SELECT
  channel,
  COUNT(*) as visits,
  COUNT(CASE WHEN is_lead THEN 1 END) as conversions,
  ROUND(100.0 * COUNT(CASE WHEN is_lead THEN 1 END) / COUNT(*), 2) as conversion_rate
FROM attribution_data
WHERE created_at >= NOW() - INTERVAL '30 days'
GROUP BY channel
ORDER BY visits DESC;

-- Spam detection effectiveness
SELECT
  classification,
  COUNT(*) as count,
  ROUND(AVG(spam_score), 2) as avg_score
FROM spam_submissions
GROUP BY classification
ORDER BY count DESC;
```

### Common Issues and Solutions

#### Issue: Workflow timeout

**Solution:** Increase timeout in HTTP Request nodes to 90000ms (90 seconds)

#### Issue: Redis connection refused

**Solution:**
```bash
# Verify Redis is running
kubectl get pods -n database | grep redis

# Test connection
redis-cli -h your-redis-host ping

# Check n8n Redis credential
```

#### Issue: Social media posts failing

**Solution:**
1. Check OAuth2 tokens haven't expired
2. Re-authenticate in n8n credentials
3. Verify API permissions for each platform

#### Issue: GitHub commits failing

**Solution:**
1. Verify GitHub Personal Access Token
2. Check token has `repo` scope
3. Verify repository exists and token has access

### Backup PostgreSQL

```bash
# Full database backup
pg_dump -h your-host -U postgres postgres > hds-backup-$(date +%Y%m%d).sql

# Restore from backup
psql -h your-host -U postgres postgres < hds-backup-20251110.sql
```

### Backup Qdrant Collections

```bash
# Create snapshot
curl -X POST http://qdrant-service:6333/collections/leads/snapshots

# Download snapshot
curl -X GET http://qdrant-service:6333/collections/leads/snapshots/{snapshot_name} \
  -o qdrant-leads-backup.snapshot

# Restore from snapshot
curl -X PUT http://qdrant-service:6333/collections/leads/snapshots/upload \
  -F 'snapshot=@qdrant-leads-backup.snapshot'
```
